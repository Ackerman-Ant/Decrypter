JSON structure this tool expects
{ "entries": [ { "category": "Logins", "title": "Gmail", "fields": [ { "label": "Username", "value": "you@example.com", "sensitive": false }, { "label": "Password", "value": "your-real-password" } ], "notes": "optional notes, shown unmasked" }, { "category": "Authenticator", "title": "GitHub 2FA", "fields": [ { "label": "Secret Key", "value": "JBSWY3DPEHPK3PXP" } ] }, { "category": "Crypto Wallets", "title": "Ledger Backup", "fields": [ { "label": "Seed Phrase", "value": "your 24 word phrase here" } ] } ] }

• "category" is any text you choose — it becomes a filter tab automatically.
• "fields" is a list of label/value pairs shown (and individually copyable) on each card.
• Add "sensitive": false to a field to show it in plain text (e.g. usernames, URLs). Omit it, or set true, to mask it behind dots until revealed.
• "notes" is optional and always shown unmasked.
• If your JSON doesn't have a top-level "entries" array, the entries will not be displayed.

Compatible with files encrypted via openssl enc -aes-256-cbc -pbkdf2 -salt -in file.json -out file.json.enc. If you used a custom -iter or -md flag, set them in the Advanced section above.
